odin4

Modern Samsung firmware flashing tool for Linux

A clean, correct, and robust implementation of the Thor USB protocol for Linux, providing a reliable alternative for Samsung device maintenance and development workflows.

Core Features

Automatic Detection

Automatic detection of Samsung devices in Download Mode via libusb

Native Thor Protocol

Custom native implementation of the Thor protocol for efficient and accurate communication

Automatic PIT Retrieval

Automatically retrieves and parses Partition Information Table (PIT) data from devices

Strict Partition Validation

Ensures firmware partitions match the device's PIT, preventing incompatibilities

Multiple Component Support

Compatible with BL, AP, CP, CSC, and UMS - all standard Samsung firmware components

Integrity Verification

Verifies .tar.md5 file integrity using Crypto++ to ensure firmware is not corrupted

LZ4 Compression Support

Supports LZ4-compressed images for faster transfers

Progress Reporting

Provides detailed per-file progress with real-time transfer statistics

Safe Dry-run Mode

--check-only option validates without flashing, ensuring complete safety

Multi-device Flashing

Supports sequential flashing of multiple devices when no specific device is targeted

Persistent Logging

Generates odin4.log file for post-operation analysis and debugging

Model Compatibility Check

Verifies firmware compatibility with the connected device model

Safety and Validation

odin4 prioritizes device safety by enforcing strict checks before any write operations:

  • Firmware Integrity Verification: Ensures firmware is not corrupted or tampered with
  • Partition Validation: Prevents flashing incorrect partitions to wrong locations
  • Compatibility Check: Validates firmware compatibility with device model
  • Explicit Control Flow: Requires clear commands for flashing, reducing accidental operations
  • Optional Dry-run: Allows simulating the flashing process without risk

The tool is designed to be non-destructive and explicitly avoids high-risk operations like NAND erase or forced repartitioning that could cause irreversible device damage.

Getting Started

Dependencies

Ensure the following packages are installed on your system:

sudo apt-get update sudo apt-get install -y cmake ninja-build zip make pkg-config g++ libusb-1.0-0-dev libcrypto++-dev

Building from Source

Follow the steps below to compile odin4 on your Linux system:

1. Clone the repository

git clone https://github.com/Llucs/odin4.git cd odin4

2. Compile the project

cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release cmake --build build --parallel $(nproc)

After successful compilation, the odin4 executable will be located in the build/ directory.

Linux USB Permissions

If odin4 reports LIBUSB_ERROR_ACCESS, you need to install the udev rule provided:

1. Copy the udev rule

sudo cp udev/60-odin4.rules /etc/udev/rules.d/60-odin4.rules

2. Reload udev rules

sudo udevadm control --reload-rules sudo udevadm trigger

3. Reconnect your device

Disconnect and reconnect your Samsung device (in Download Mode) to apply the new permissions.

Documentation

Command-Line Options

Usage: odin4 [options] Samsung firmware flashing tool for Linux. Version: x.x.x-xxxxxx Options: -h Show this help message -v Show version -w Show license -l List detected Download Mode devices -d <path> Select a specific USB device path (e.g. /dev/bus/usb/001/002) -b <file> Bootloader archive (.tar or .tar.md5) -a <file> AP archive (.tar or .tar.md5) -c <file> CP archive (.tar or .tar.md5) -s <file> CSC archive (.tar or .tar.md5) -u <file> UMS archive (.tar or .tar.md5) --check-only Validate PIT + archives and exit without flashing --allow-unknown Allow archive entries without a PIT match (disabled by default) --reboot Reboot device after flashing --redownload Reboot into download mode if supported Logging: --quiet Only print errors --verbose More detailed logs --debug Debug logs (includes USB packet hexdumps) USB selection overrides (optional): --vid <hex> Override USB vendor ID (hex, e.g. 04e8) --pid <hex> Override USB product ID (hex) --usb-interface <n> Force a specific USB interface number

Usage Examples

List devices in Download Mode

./build/odin4 -l

Flash firmware (all components)

./build/odin4 \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5 \ -c CP_XXXX.tar.md5 \ -s CSC_XXXX.tar.md5

Flash specific device with selected components

./build/odin4 -d /dev/bus/usb/001/002 \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Perform dry-run (validation only)

./build/odin4 --check-only \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Reboot device after flashing

./build/odin4 --reboot \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Library Documentation

odin4 provides a C++ library (libodin4) for integration into other tools and applications. The library is available in both static (.a) and shared (.so) formats.

Main Structures

OdinConfig: Defines configuration parameters for odin4 operations, including firmware paths, dry-run options, logging, and USB device selection.

OdinExitCode: Enumeration representing possible exit codes (Success, Usage, Usb, Protocol, Pit, Firmware, Unknown).

Core Functions

  • void odin4_init(const OdinConfig& cfg) - Initializes the library
  • std::vector<std::string> odin4_list_devices(const OdinConfig& cfg) - Lists detected devices
  • OdinExitCode odin4_run(const OdinConfig& cfg) - Executes flashing operation
  • const char* odin4_get_version() - Returns library version

Library Dependencies

  • libusb-1.0: Low-level USB communication with devices
  • Crypto++: Cryptographic operations and MD5 verification
  • LZ4: LZ4-compressed data decompression

Compile Application with libodin4

g++ my_app.cpp -o my_app -I./include -L./lib -lodin4 -lusb-1.0 -lcryptopp -lpthread -ldl

Supported Firmware Components

BL - Bootloader

Device initialization code responsible for loading the kernel and operating system

AP - Application Processor

Main processor and operating system (Android/Linux), contains kernel and system applications

CP - Modem/Phone

Communication modem firmware responsible for cellular connectivity and radio

CSC - Consumer Software Customization

Region and carrier-specific customizations, includes pre-installed applications

UMS - USB Mass Storage

USB storage firmware (less common in modern devices)
