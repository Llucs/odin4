Thor Protocol

Understand the engineering behind Samsung device communication.

What is the Thor Protocol?

The Thor protocol is the proprietary language used by Samsung for communication between the host (PC) and the device in Download Mode. It operates over USB and handles critical tasks like partition table delivery and firmware writing.

Technical Note: odin4 implements this protocol natively, ensuring every packet follows the exact specifications expected by the device bootloader.

Session Start (Handshake)

Before any operation, the host must initiate a session. This is done by sending an initialization packet.

Value Type Description
0x4F44494E String ("ODIN") Magic Identification Number
0x0001 16-bit Integer Protocol Version

PIT Table Extraction

The Partition Information Table (PIT) defines the device's storage layout. odin4 requests this table to validate firmware compatibility.

PIT Request Command (0x65)
Packet: [0x65] [0x00] [0x00] [0x00] ...

The device responds with the PIT size followed by the raw binary data.

Flash Sequence

Flashing occurs in three main phases for each file:

  1. Start (0x66): Notifies the device about the file name and size.
  2. Data (0x67): Sending data chunks (usually 1MB or 128KB).
  3. End (0x66 + 0x03/0x07): Finalizes writing and requests checksum verification.

Device Metadata (0x69)

Allows extracting vital information without the need for flashing.

  • 0x00: Model Name (e.g., SM-G991B)
  • 0x01: Serial Number
  • 0x02: Region Code (CSC)
  • 0x03: Carrier ID

Getting Started

Follow the steps below to set up odin4 on your system.

01

Dependencies

Install the required libraries via terminal:

sudo apt-get update sudo apt-get install -y cmake ninja-build zip make pkg-config g++ libusb-1.0-0-dev libcrypto++-dev
02

Compilation

Clone and compile the project using CMake:

git clone https://github.com/Llucs/odin4.git cd odin4 cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release cmake --build build --parallel $(nproc)
03

USB Permissions

Configure udev rules for non-root access:

sudo cp udev/60-odin4.rules /etc/udev/rules.d/ sudo udevadm control --reload-rules sudo udevadm trigger

Usage Documentation

odin4 --help 
Usage: odin4 [options]
Samsung flashing tool for Linux.

Options:
  -l                  List detected devices
  -b <file>           Bootloader file (.tar.md5)
  -a <file>           AP file (.tar.md5)
  -c <file>           CP file (.tar.md5)
  -s <file>           CSC file (.tar.md5)
  --check-only        Validate files only
  --reboot            Reboot after flashing
  --verbose           Detailed logs
Example: Full Flash
./odin4 -b BL.tar.md5 -a AP.tar.md5 -c CP.tar.md5 -s CSC.tar.md5 --reboot
Example: Verification Only
./odin4 --check-only -a AP.tar.md5
