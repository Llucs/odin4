Protocolo Thor

Entenda a engenharia por trás da comunicação com dispositivos Samsung.

O que é o Protocolo Thor?

O protocolo Thor é a linguagem proprietária utilizada pela Samsung para comunicação entre o host (PC) e o dispositivo em Download Mode. Ele opera sobre USB e é responsável por tarefas críticas como envio de tabelas de partição e escrita de firmware.

Nota Técnica: O odin4 implementa este protocolo de forma nativa, garantindo que cada pacote enviado siga exatamente as especificações esperadas pelo bootloader do dispositivo.

Início da Sessão (Handshake)

Antes de qualquer operação, o host deve iniciar uma sessão. Isso é feito enviando um pacote de inicialização.

Valor Tipo Descrição
0x4F44494E String ("ODIN") Número Mágico de Identificação
0x0001 Inteiro 16-bit Versão do Protocolo

Extração da Tabela PIT

A Partition Information Table (PIT) define o layout de armazenamento do dispositivo. O odin4 solicita esta tabela para validar se o firmware é compatível.

Comando de Requisição PIT (0x65)
Pacote: [0x65] [0x00] [0x00] [0x00] ...

O dispositivo responde com o tamanho da PIT seguido pelos dados binários brutos.

Sequência de Flash

O flash ocorre em três fases principais para cada arquivo:

  1. Início (0x66): Notifica o dispositivo sobre o nome e tamanho do arquivo.
  2. Dados (0x67): Envio de chunks de dados (geralmente 1MB ou 128KB).
  3. Fim (0x66 + 0x03/0x07): Finaliza a escrita e solicita a verificação de checksum no dispositivo.

Metadados do Dispositivo (0x69)

Permite extrair informações vitais sem necessidade de flash.

  • 0x00: Nome do Modelo (ex: SM-G991B)
  • 0x01: Número de Série
  • 0x02: Código da Região (CSC)
  • 0x03: ID da Operadora

Começando

Siga os passos abaixo para configurar o odin4 no seu sistema.

01

Dependências

Instale as bibliotecas necessárias via terminal:

sudo apt-get update sudo apt-get install -y cmake ninja-build zip make pkg-config g++ libusb-1.0-0-dev libcrypto++-dev
02

Compilação

Clone e compile o projeto usando CMake:

git clone https://github.com/Llucs/odin4.git cd odin4 cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release cmake --build build --parallel $(nproc)
03

Permissões USB

Configure as regras udev para acesso sem root:

sudo cp udev/60-odin4.rules /etc/udev/rules.d/ sudo udevadm control --reload-rules sudo udevadm trigger

Documentação de Uso

odin4 --help 
Uso: odin4 [opções]
Ferramenta de flash Samsung para Linux.

Opções:
  -l                  Listar dispositivos detectados
  -b <arquivo>        Arquivo Bootloader (.tar.md5)
  -a <arquivo>        Arquivo AP (.tar.md5)
  -c <arquivo>        Arquivo CP (.tar.md5)
  -s <arquivo>        Arquivo CSC (.tar.md5)
  --check-only        Apenas validar arquivos
  --reboot            Reiniciar após o flash
  --verbose           Logs detalhados
Exemplo: Flash Completo
./odin4 -b BL.tar.md5 -a AP.tar.md5 -c CP.tar.md5 -s CSC.tar.md5 --reboot
Exemplo: Apenas Verificação
./odin4 --check-only -a AP.tar.md5
