Thor Protocol

Understand the engineering behind Samsung device communication.

What is the Thor Protocol?

The Thor protocol is the proprietary language used by Samsung for communication between the host (PC) and the device in Download Mode. It operates over USB and handles critical tasks like partition table delivery and firmware writing.

Technical Note: odin4 implements this protocol natively, ensuring every packet follows the exact specifications expected by the device bootloader.

Session Start (Handshake)

Before any operation, the host must initiate a session by sending a handshake packet with the magic identifier and protocol version.

Offset Size Field Description
0x00 4 bytes packet_size Total packet size (0x000C)
0x04 2 bytes packet_type 0x0001 = HANDSHAKE
0x06 2 bytes packet_flags Flags word
0x08 4 bytes magic 0x4F44494E ("ODIN")
0x0C 4 bytes version Protocol version

PIT Table Extraction

The Partition Information Table (PIT) defines the device's storage layout. odin4 requests this table to validate firmware compatibility.

Request PIT (Packet Type 0x0007)
ThorPitFilePacket: packet_size = 0x0C packet_type = 0x0007 (THOR_PACKET_PIT_FILE) pit_file_size = 0x00000000

The device responds with PIT data using the same packet type.

Flash Sequence

Flashing occurs in three main phases for each file using these packet types:

Phase Packet Type Name Description
Begin 0x0008 BEGIN_SESSION Start file transfer session
Begin 0x0009 FILE_PART_SIZE Notify file size
Data 0x0003 FILE_PART Sending data chunks
Data 0x000A RECEIVE_FILE_PART Continue data transfer
End 0x0004 END_FILE_TRANSFER Finalize and verify

Device Information

Query device metadata using packet type 0x0002 (DEVICE_TYPE) for device information retrieval.

Field Size Description
device_type 128 bytes Null-terminated string (e.g., "SM-G991B")

Control Operations

Device control using packet type 0x000B (CONTROL):

Control Type Value Description
REBOOT 0x0001 Reboot device normally
REDOWNLOAD 0x0002 Reboot into download mode

Getting Started

Follow the steps below to set up odin4 on your system.

01

Dependencies

Install the required libraries via terminal:

sudo apt-get update sudo apt-get install -y cmake ninja-build zip make pkg-config g++ libusb-1.0-0-dev libcrypto++-dev
02

Compilation

Clone and compile the project using CMake:

git clone https://github.com/Llucs/odin4.git cd odin4 cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release cmake --build build --parallel $(nproc)
03

USB Permissions

Configure udev rules for non-root access:

sudo cp udev/60-odin4.rules /etc/udev/rules.d/ sudo udevadm control --reload-rules sudo udevadm trigger

Usage Documentation

odin4 --help 
Usage: odin4 [options]
Samsung firmware flashing tool for Linux. Version: x.x.x

Options:
  -h                  Show this help message
  -v                  Show version
  -w                  Show license
  -l                  List detected Download Mode devices
  -d <path>            Select a specific USB device path
  -b <file>           Bootloader archive (.tar or .tar.md5)
  -a <file>           AP archive (.tar or .tar.md5)
  -c <file>           CP archive (.tar or .tar.md5)
  -s <file>           CSC archive (.tar or .tar.md5)
  -u <file>           UMS archive (.tar or .tar.md5)
  --check-only         Validate PIT + archives and exit without flashing
  --allow-unknown      Allow archive entries without a PIT match
  --reboot             Reboot device after flashing
  --redownload         Reboot into download mode if supported

Logging:
  --quiet              Only print errors
  --verbose            More detailed logs
  --debug              Debug logs (includes USB packet hexdumps)

USB selection overrides:
  --vid <hex>          Override USB vendor ID
  --pid <hex>          Override USB product ID
  --usb-interface <n>  Force a specific USB interface number
Example: Full Flash
./odin4 -b BL.tar.md5 -a AP.tar.md5 -c CP.tar.md5 -s CSC.tar.md5 -u UMS.tar.md5 --reboot
Example: Specific Device
./odin4 -d /dev/bus/usb/001/002 -a AP.tar.md5 --check-only
Example: Redownload Mode
./odin4 -a AP.tar.md5 --reboot --redownload
