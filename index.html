odin4

Ferramenta moderna de flash para Samsung no Linux

Uma implementação limpa, correta e robusta do protocolo Thor USB para Linux, oferecendo uma alternativa confiável para manutenção de dispositivos Samsung e fluxos de trabalho de desenvolvimento.

Recursos Principais

Detecção Automática

Detecção automática de dispositivos Samsung em Download Mode via libusb

Protocolo Thor Nativo

Implementação nativa e customizada do protocolo Thor para comunicação eficiente e precisa

Recuperação Automática de PIT

Recupera e analisa automaticamente dados da Tabela de Informações de Partição (PIT)

Validação Rigorosa de Partições

Garante que partições de firmware correspondam ao PIT do dispositivo, prevenindo incompatibilidades

Suporte a Múltiplos Componentes

Compatível com BL, AP, CP, CSC e UMS - todos os componentes padrão de firmware Samsung

Verificação de Integridade

Verifica integridade de arquivos .tar.md5 usando Crypto++ para garantir firmware não corrompido

Suporte a Compressão LZ4

Suporta imagens comprimidas em LZ4 para transferências mais rápidas

Relatório de Progresso

Fornece progresso detalhado por arquivo com estatísticas de transferência em tempo real

Modo Dry-run Seguro

Opção --check-only para validar sem realizar flash, garantindo segurança total

Flash Múltiplos Dispositivos

Suporta flash sequencial de múltiplos dispositivos quando nenhum é especificado

Arquivo de Log Persistente

Gera arquivo odin4.log para análise pós-operação e debugging

Compatibilidade de Modelo

Verifica compatibilidade de firmware com o modelo do dispositivo conectado

Segurança e Validação

odin4 prioriza a segurança do dispositivo aplicando verificações rigorosas antes de qualquer operação de escrita:

  • Verificação de Integridade: Garante que o firmware não está corrompido ou adulterado
  • Validação de Partição: Previne flash de partições incorretas para locais errados
  • Verificação de Compatibilidade: Valida compatibilidade do firmware com o modelo do dispositivo
  • Fluxo de Controle Explícito: Requer comandos claros para flash, reduzindo operações acidentais
  • Execução Dry-run Opcional: Permite simular o processo de flash sem risco

A ferramenta é projetada para ser não-destrutiva e evita explicitamente operações de alto risco como apagamento NAND ou reparticionamento forçado.

Começando

Dependências

Certifique-se de que os seguintes pacotes estão instalados no seu sistema:

sudo apt-get update sudo apt-get install -y cmake ninja-build zip make pkg-config g++ libusb-1.0-0-dev libcrypto++-dev

Compilar a partir do Código-Fonte

Siga os passos abaixo para compilar odin4 no seu sistema Linux:

1. Clonar o repositório

git clone https://github.com/Llucs/odin4.git cd odin4

2. Compilar o projeto

cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release cmake --build build --parallel $(nproc)

Após compilação bem-sucedida, o executável odin4 estará localizado no diretório build/.

Permissões USB no Linux

Se odin4 reportar LIBUSB_ERROR_ACCESS, você precisa instalar a regra udev fornecida:

1. Copiar a regra udev

sudo cp udev/60-odin4.rules /etc/udev/rules.d/60-odin4.rules

2. Recarregar regras udev

sudo udevadm control --reload-rules sudo udevadm trigger

3. Reconectar seu dispositivo

Desconecte e reconecte seu dispositivo Samsung (em Download Mode) para aplicar as novas permissões.

Documentação

Opções de Linha de Comando

Usage: odin4 [options] Samsung firmware flashing tool for Linux. Version: x.x.x-xxxxxx Options: -h Show this help message -v Show version -w Show license -l List detected Download Mode devices -d <path> Select a specific USB device path (e.g. /dev/bus/usb/001/002) -b <file> Bootloader archive (.tar or .tar.md5) -a <file> AP archive (.tar or .tar.md5) -c <file> CP archive (.tar or .tar.md5) -s <file> CSC archive (.tar or .tar.md5) -u <file> UMS archive (.tar or .tar.md5) --check-only Validate PIT + archives and exit without flashing --allow-unknown Allow archive entries without a PIT match (disabled by default) --reboot Reboot device after flashing --redownload Reboot into download mode if supported Logging: --quiet Only print errors --verbose More detailed logs --debug Debug logs (includes USB packet hexdumps) USB selection overrides (optional): --vid <hex> Override USB vendor ID (hex, e.g. 04e8) --pid <hex> Override USB product ID (hex) --usb-interface <n> Force a specific USB interface number

Exemplos de Uso

Listar dispositivos em Download Mode

./build/odin4 -l

Flash de firmware (todos os componentes)

./build/odin4 \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5 \ -c CP_XXXX.tar.md5 \ -s CSC_XXXX.tar.md5

Flash de dispositivo específico com componentes selecionados

./build/odin4 -d /dev/bus/usb/001/002 \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Realizar dry-run (validação apenas)

./build/odin4 --check-only \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Rebootar dispositivo após flash

./build/odin4 --reboot \ -b BL_XXXX.tar.md5 \ -a AP_XXXX.tar.md5

Documentação da Biblioteca

odin4 fornece uma biblioteca C++ (libodin4) para integração em outras ferramentas e aplicações. A biblioteca está disponível em formatos estático (.a) e compartilhado (.so).

Estruturas Principais

OdinConfig: Define parâmetros de configuração para operações odin4, incluindo caminhos de firmware, opções de dry-run, logging e seleção de dispositivo USB.

OdinExitCode: Enumeração representando possíveis códigos de saída (Success, Usage, Usb, Protocol, Pit, Firmware, Unknown).

Funções Principais

  • void odin4_init(const OdinConfig& cfg) - Inicializa a biblioteca
  • std::vector<std::string> odin4_list_devices(const OdinConfig& cfg) - Lista dispositivos detectados
  • OdinExitCode odin4_run(const OdinConfig& cfg) - Executa operação de flash
  • const char* odin4_get_version() - Retorna versão da biblioteca

Dependências da Biblioteca

  • libusb-1.0: Comunicação USB de baixo nível com dispositivos
  • Crypto++: Operações criptográficas e verificação MD5
  • LZ4: Descompressão de dados comprimido em LZ4

Compilar Aplicação com libodin4

g++ my_app.cpp -o my_app -I./include -L./lib -lodin4 -lusb-1.0 -lcryptopp -lpthread -ldl

Componentes de Firmware Suportados

BL - Bootloader

Código de inicialização do dispositivo, responsável pelo carregamento do kernel e sistema operacional

AP - Application Processor

Processador principal e sistema operacional (Android/Linux), contém o kernel e aplicações do sistema

CP - Modem/Phone

Firmware do modem de comunicação, responsável por conectividade celular e rádio

CSC - Consumer Software Customization

Customizações específicas de região e operadora, inclui aplicações pré-instaladas

UMS - USB Mass Storage

Firmware de armazenamento USB (menos comum em dispositivos modernos)
